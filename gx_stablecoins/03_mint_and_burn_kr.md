# 발행 및 소각

GX Stablecoin 네트워크는 은행 API 이벤트에 직접 고정된 자동화된 발행 및 소각 메커니즘을 사용하여 디지털 공급이 항상 실제 법정화폐 준비금에 해당하도록 보장합니다. 이 페이지는 발행 및 소각 프로세스, 은행 API 통합 및 1:1 뒷받침 비율을 유지하기 위한 멱등성 보호를 자세히 설명합니다.

## 발행 프로세스

사용자가 법정화폐를 GX 스테이블코인으로 변환하려는 경우:

1. **법정화폐 예금**: 사용자가 규제된 은행 계정에 법정화폐를 예금합니다
2. **은행 API 알림**: 은행 시스템이 GX 체인 게이트웨이에 예금 이벤트를 알립니다
3. **발행 요청 생성**: 게이트웨이가 고유 ID로 발행 요청을 생성합니다
4. **정족수 승인**: 라이선스 검증자가 예금 및 규정 준수를 확인합니다
5. **멱등성 확인**: 체인이 중복 처리를 방지하기 위해 요청이 고유한지 확인합니다
6. **발행 실행**: 스테이블코인이 발행되어 사용자 지갑에 적립됩니다
7. **준비금 증명 업데이트**: 준비금 잔액이 자동으로 업데이트됩니다

### 주요 기능:

- **자동화**: 수동 개입 없이 완전 자동화된 프로세스
- **멱등성**: 안전한 재시도로 이중 발행 방지
- **규정 준수**: 모든 발행에 대한 KYC/AML 확인 내장
- **투명성**: 모든 발행 이벤트가 온체인에 기록됨

## 소각 프로세스

사용자가 스테이블코인을 법정화폐로 상환하려는 경우:

1. **상환 요청**: 사용자가 소각 요청을 시작합니다
2. **정족수 승인**: 검증자가 규정 준수 및 준비금을 확인합니다
3. **멱등성 확인**: 체인이 요청이 고유한지 확인합니다
4. **소각 실행**: 스테이블코인이 사용자 지갑에서 차감됩니다
5. **법정화폐 해제**: 은행 API가 법정화폐 지불을 트리거합니다
6. **사용자 지불**: 은행이 사용자에게 법정화폐를 전송합니다

### 보호 장치:

- **멱등성**: 중복 소각으로부터 보호
- **준비금 확인**: 법정화폐가 해제되기 전에 소각이 확인됨
- **결정적 최종성**: 1-3초 내에 거래 최종성
