# 발행 및 소각 메커니즘

발행 및 소각 메커니즘은 GX Stablecoin Chain의 기초 구성 요소로, 실제 법정화폐 준비금에 고정된 주권 스테이블코인의 자동화된 발행 및 상환을 가능하게 합니다. 이 프로세스는 안전한 API를 통해 라이선스된 은행 파트너와 긴밀하게 통합되어 발행되거나 소각된 모든 토큰이 은행 시스템의 실제 법정화폐 이동에 해당하도록 보장합니다. 이 페이지는 자동화된 발행 및 소각 워크플로에 대한 포괄적인 개요를 제공하고 멱등성, 정족수 승인 및 결정적 최종성과 같은 중요한 개념을 설명하며 개발자와 통합자를 안내하기 위한 자세한 프로세스 흐름 표를 제시합니다.

***

## 개요

GX Stablecoin Chain은 특정 관할권에 바인딩된 라이선스 검증자가 있는 허가된 Proof-of-Authority(PoA) 합의 모델을 운영합니다. GX 체인에서 발행된 스테이블코인(예: USGX, KRGX, JPGX, PHGX)은 1:1 비율로 완전히 법정화폐로 뒷받침되며 준비금은 규제된 은행에 보관되고 지속적인 준비금 증명 투명성의 대상이 됩니다.

스테이블코인의 발행 및 소각은 자동화되고 은행 API 이벤트에 고정되어 토큰 공급이 오프체인 법정화폐 준비금의 온체인 표현에 엄격하게 해당하도록 보장합니다. 이 통합은 수동 조정 지연을 제거하고 이중 발행 또는 상환과 관련된 위험을 완화합니다.

***

## 자동화된 발행 및 소각 프로세스

발행 및 소각 프로세스는 은행 API를 통해 감지된 법정화폐 이동에 의해 트리거됩니다. 사용자가 관리 은행 계정에 법정화폐를 예금하면 GX 체인에서 자동화된 발행 이벤트가 시작되어 사용자에게 동등한 스테이블코인 금액이 적립됩니다. 반대로 사용자가 스테이블코인을 상환하면 소각 프로세스가 시작되고 해당 법정화폐 금액이 은행에서 사용자에게 해제됩니다.

| 5 | **발행 실행**: 정족수 승인 및 멱등성 검증 후 발행 트랜잭션이 온체인에서 실행됩니다. | GX Chain 검증자 | 스테이블코인이 발행되어 사용자 지갑에 적립됩니다. |
| 6 | **최종성 확인**: 결정적 최종성은 발행 트랜잭션이 1초 미만 내에 되돌릴 수 없고 최종화됨을 보장합니다. | GX Chain 합의 레이어 | 발행 트랜잭션이 최종화되고 기록됩니다. |
| 7 | **법정화폐 상환 요청**: 사용자가 스테이블코인 상환을 시작하여 소각 요청을 트리거합니다. | 사용자 & GX Chain Gateway | 소각 요청이 생성되고 대기열에 추가됩니다. |
| 8 | **정족수 승인**: 검증자가 규정 준수 및 준비금 확인을 기반으로 소각 요청을 검토하고 승인합니다. | 라이선스 검증자 (PoA) | 소각 요청이 승인됩니다. |
| 9 | **멱등성 확인**: 시스템이 소각 요청이 고유하고 이전에 처리되지 않았는지 확인합니다. | GX Chain 모듈 | 중복 소각 요청이 거부됩니다. |
| 10 | **소각 실행**: 스테이블코인이 온체인에서 소각되어 순환 공급을 줄입니다. | GX Chain 검증자 | 토큰이 소각되고 사용자 잔액이 차감됩니다. |
| 11 | **법정화폐 해제**: 은행이 사용자에게 법정화폐 지불을 처리하여 상환 주기를 완료합니다. | 은행 API & 관리 은행 | 법정화폐 자금이 사용자의 은행 계정으로 전송됩니다. |

## 주요 개념 설명

### 멱등성

멱등성은 동일한 발행 또는 소각 요청의 반복 처리가 여러 토큰 발행 또는 상환을 초래하지 않도록 보장하는 중요한 설계 원칙입니다. 이것은 네트워크 재시도 또는 중복 API 호출이 발생할 수 있는 분산 시스템에서 필수적입니다. GX 체인은 각 은행 이벤트 및 발행/소각 요청에 고유 식별자를 할당하여 멱등성을 구현합니다. 처리 전에 시스템은 요청 ID가 이미 실행되었는지 확인하여 이중 지출 또는 이중 발행을 방지합니다.

### 정족수 승인

보안 및 규제 준수를 유지하기 위해 발행 및 소각 요청은 Proof-of-Authority 합의 하에 운영되는 라이선스 검증자 정족수의 다자 승인이 필요합니다. 이 정족수 메커니즘은 직무 분리를 시행하고 일방적인 발행 또는 상환을 방지합니다. 검증자는 거래에 서명하기 전에 은행 이벤트의 정당성, 규정 준수 상태 및 준비금 충분성을 확인합니다. 필요한 정족수 임계값에 도달해야만 시스템이 발행 또는 소각을 진행합니다.

### 결정적 최종성

결정적 최종성은 발행 또는 소각 트랜잭션이 GX 체인에 커밋되면 되돌릴 수 없고 모든 네트워크 참여자에 의해 즉시 인식됨을 보장합니다. Tendermint 클래스 Byzantine Fault Tolerant 합의를 활용하여 GX 체인은 1초 미만의 최종성을 달성하여 스테이블코인 발행 및 상환의 신속하고 예측 가능한 정산을 가능하게 합니다. 이 결정적 최종성은 온체인 토큰 공급과 오프체인 법정화폐 준비금 간의 신뢰 및 동기화를 유지하는 데 중요합니다.

## 프로세스 흐름 표

### 발행 프로세스 흐름

| 단계 | 설명 | 시스템 구성 요소 | 참고 사항 |
| ------------------ | --------------------------------------------------------------------------------------- | ----------------------- | ------------------------------------- |
| 예금 감지 | 은행 API가 법정화폐 예금을 감지하고 이벤트 알림을 보냅니다. | 은행 API | 이벤트에는 고유 트랜잭션 ID가 포함됩니다. |
| 요청 생성 | GX 체인 게이트웨이가 은행 이벤트 ID를 참조하는 발행 요청을 생성합니다. | GX Chain Gateway 모듈 | 트랜잭션 페이로드를 준비합니다. |
| 검증자 검토 | 라이선스 검증자가 예금 진위성 및 규정 준수를 확인합니다. | PoA 검증자 | 다중 서명 정족수가 필요합니다. |
| 멱등성 확인 | 시스템이 발행 요청 ID가 이미 처리되었는지 확인합니다. | GX Chain Mint 모듈 | 이중 발행을 방지합니다. |
| 발행 실행 | 발행 트랜잭션이 온체인에서 실행되어 사용자 지갑에 스테이블코인을 적립합니다. | GX Chain 검증자 | 트랜잭션이 브로드캐스트되고 커밋됩니다. |
| 최종성 | 트랜잭션이 결정적 최종성에 도달하여 발행 완료를 확인합니다. | Tendermint 합의 | 1초 미만 최종성. |
| 확인 | 사용자 지갑 잔액이 업데이트되고 감사 및 준비금 증명 투명성을 위해 이벤트가 기록됩니다. | GX Chain 원장 | 실시간 준비금 증명이 업데이트됩니다. |

| 검증자 검토 | 검증자가 소각 요청 규정 준수, 준비금 충분성 및 사용자 승인을 확인합니다. | PoA 검증자 | 다중 서명 정족수가 필요합니다. |
| 멱등성 확인 | 시스템이 소각 요청이 이전에 처리되지 않았는지 확인합니다. | GX Chain Burn 모듈 | 이중 소각을 방지합니다. |
| 소각 실행 | 스테이블코인이 온체인에서 소각되어 순환 공급을 줄입니다. | GX Chain 검증자 | 트랜잭션이 브로드캐스트되고 커밋됩니다. |
| 최종성 | 소각 트랜잭션이 결정적 최종성에 도달하여 토큰 파괴를 확인합니다. | Tendermint 합의 | 1초 미만 최종성. |
| 법정화폐 해제 | 은행 API가 상환 요청에 따라 사용자의 은행 계정으로 법정화폐 해제를 트리거합니다. | 은행 API & 관리 은행 | 오프체인 법정화폐 이동이 주기를 완료합니다. |

## 요약

GX Stablecoin Chain의 발행 및 소각 메커니즘은 법정화폐와 블록체인 자산을 연결하는 강력하고 안전하며 규정을 준수하는 접근 방식을 예시합니다. 토큰 발행 및 상환을 은행 API 이벤트에 고정하고 멱등성을 시행하며 정족수 승인을 요구하고 결정적 최종성을 활용함으로써 시스템은 스테이블코인 공급이 항상 완전히 뒷받침되고 실제 법정화폐 준비금을 정확하게 반영하도록 보장합니다. 이 메커니즘은 GX 스테이블코인의 신뢰성과 규제 정렬을 뒷받침하여 Gurufin Chain 생태계에서 원활한 국경 간 결제, FX 거래 및 DeFi 애플리케이션을 가능하게 합니다.
