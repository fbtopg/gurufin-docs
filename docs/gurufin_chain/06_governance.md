# Governance

Gurufin Chain utilizes on-chain governance powered by the DPoS consensus model. Validators and delegators use staked GXN to vote on parameter changes, protocol upgrades, and treasury allocations.

**Voting Parameters**
* **Voting Window:** 14 days.
* **Quorum Requirement:** 33.4% of bonded voting power.
* **Pass Thresholds:** >50% for standard parameter changes; ≥66.7% supermajority for economic changes or upgrades.
* **Execution:** Immediate upon passing. Gurufin Chain does not utilize timelocks.
* **Veto:** A "No with Veto" vote by ≥33.4% of the network rejects a proposal outright.

**Emergency Controls**
A Security Council (n-of-m multisig) possesses narrowly scoped emergency powers (e.g., freezing pairs during oracle failure). These actions automatically expire within 72 hours unless formally ratified by a token holder vote.
